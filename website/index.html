00:00:00  Subtitles MD

Read .srt and .vtt transcripts like prose.

Subtitles MD renders subtitle files as searchable Markdown in Obsidian. No playback required, just comfortable, linkable text.

Built for researchers, students, and note-takers who keep lectures, interviews, and talks in their vault.

00:00:24features

A reading tool, not a media player

Four things Subtitles MD does well, so transcripts behave like every other note in your vault.

01 / convert

One command to a clean note

Run Convert subtitle file to transcript note on any .srt or .vtt file. Subtitles MD reflows the cues into readable paragraphs and writes a new Markdown note beside the original. The source file stays untouched.

⌘/Ctrl + P→Convert subtitle file to transcript note

02 / embed

A transcript code block

Drop a ```transcript block into any note. Paste subtitle text inline, or point at a file with file: path/to/sub.srt. It renders as paragraphs in reading view.

03 / read

Read without converting

Click any .srt or .vtt in your vault to open it in a calm, read-only transcript view. Nothing is rewritten, nothing is generated.

04 / shape

Formatting on your terms

Decide how cues group into paragraphs and how timestamps, speakers, and frontmatter appear. Six settings, sensible defaults, and everything stays on your machine.

00:02:38faq

Questions

Which formats are supported?

.srt (SubRip) and .vtt (WebVTT), including transcripts exported from most lecture, meeting, and captioning tools.

Does it play video or audio?

No, and that is the point. Subtitles MD is reading-first: it renders subtitle text as Markdown so you can read, search, and annotate it. There is no media player to manage.

Does my data stay on my machine?

Yes. Everything runs locally inside your vault. No accounts, no servers, no network requests. Your transcripts never leave your device.

Will it change my original files?

No. The read-only view and the transcript code block leave your .srt and .vtt files untouched. Converting creates a new note beside the source.

Is it open source?

Yes, released under the MIT license. Read the source, file an issue, or send a pull request on GitHub.

How do I install it?

Build or download the release assets and drop them into your vault's plugins folder, or wait for the Community Plugins listing once review is complete. Full steps live in the README.
